## 周六做 demo 的具体策略

  做之前

  - 研究深度赋智的业务和技术栈：他们是做 AI Agent / LLM 应用方向的公司（从职位匹配度推断），搞清楚他们用什么框架、做什么产品
  - 向 HR 或 JD 要清楚 demo 的具体要求：评分标准是什么？有没有时间限制？提交格式？
  - 准备好开发环境：别到时候花一小时配环境

  做的时候

  - 核心功能优先，确保能跑通：一个稳定可演示的 MVP > 一个有5个功能但崩溃的系统
  - 代码质量要体现工程素养：不是功能多少的问题，是代码结构、命名、错误处理、README 写得清楚不清楚——面试官看的是你的思维方式和习惯
  - 留时间写 README：说明设计思路、怎么运行、有哪些 trade-off。这个往往是区分度的关键
  - 2-3小时的量，不要拖到一天：HR 说了要一气呵成，而且过度打磨反而暴露你对需求判断不果断

  做完之后

  - 提交前自己跑一遍，确保 clone 下来能直接跑
  - 给 HR 发消息确认已提交，主动推进流程
