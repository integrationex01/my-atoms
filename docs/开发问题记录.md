# 开发问题记录

## 问题 1：中文字符编码问题

**现象：** 创建项目时输入中文（如"测试项目"），数据库存储内容和前端显示内容均出现乱码。

### 原因分析

Neon PostgreSQL 数据库本身默认使用 UTF-8 编码，数据在数据库层面是正确的。问题出在两个环节：

1. **API 响应缺少 charset 声明**：Next.js 的 `NextResponse.json()` 默认设置 `Content-Type: application/json`，但**不包含 `charset=utf-8`**。浏览器可能按默认编码（如 ISO-8859-1）解析响应，导致中文乱码。

2. **DATABASE_URL 缺少客户端编码参数**：PrismaPg 适配器连接 PostgreSQL 时，未显式指定 `client_encoding=UTF8`，在特定环境下可能导致传输层编码转换异常。

### 解决方案

**修改 1：API 路由添加 charset 响应头**

涉及文件：
- `src/app/api/projects/route.ts`
- `src/app/api/projects/[id]/route.ts`
- `src/app/api/auth/register/route.ts`
- `src/app/api/auth/login/route.ts`
- `src/app/api/projects/[id]/messages/route.ts`
- `src/app/api/projects/[id]/snapshots/route.ts`

将 `NextResponse.json(data)` 改为：
```typescript
return NextResponse.json(data, {
  headers: { "Content-Type": "application/json; charset=utf-8" },
});
```

**修改 2：在 Prisma 客户端初始化时设置客户端编码**

在 `src/lib/prisma.ts` 中添加连接后执行 `SET client_encoding TO 'UTF8'`：
```typescript
const client = new PrismaClient({ adapter });
await client.$executeRawUnsafe("SET client_encoding TO 'UTF8'");
```

### 验证

修复后通过 curl 测试中文存储：
```bash
curl -s http://localhost:3000/api/projects -X POST \
  -H "Content-Type: application/json; charset=utf-8" \
  -d '{"name":"测试项目","description":"中文描述"}'
```

确认返回数据中文字符正常显示。
